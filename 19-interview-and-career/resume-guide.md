# Resume Guide

A data engineering resume should show the problems you solved, the systems you built, and the impact of your work. Tools matter, but they should support a business result.

## 1. Resume Positioning

Use a short summary that matches your target level.

Junior example:

> Data engineering learner with hands-on projects in SQL, Python, PostgreSQL, ETL pipelines, and data quality checks. Built retail-focused projects that load, clean, model, and validate business data.

Mid-level example:

> Data engineer with experience building scheduled ETL/ELT pipelines, warehouse models, dbt transformations, and data quality checks for analytics and operations use cases.

Senior example:

> Data engineer focused on reliable data platforms, warehouse and lakehouse architecture, pipeline observability, governance, and business-impact analytics systems.

## 2. Skills Section

Group skills clearly.

Example:

- Languages: SQL, Python
- Databases/Warehouses: PostgreSQL, BigQuery, Snowflake
- Pipelines: ETL/ELT, Airflow, dbt
- Data Processing: pandas, PySpark
- Quality: dbt tests, SQL checks, Great Expectations, Soda
- Cloud: AWS, GCP, Azure basics
- DevOps: Git, Docker, CI/CD basics

Only list tools you can discuss honestly.

## 3. Project Bullet Formula

Use this format:

```text
Built [system] using [tools] to [business outcome], including [reliability or quality feature].
```

Examples:

- Built a retail sales pipeline using Python, pandas, SQLAlchemy, and PostgreSQL to automate daily sales reporting, including null, duplicate, and revenue sanity checks.
- Designed a retail warehouse with fact and dimension tables to standardize product, customer, and inventory analytics across business reports.
- Created a dbt transformation project with staging, intermediate, and mart models, adding tests and documentation for trusted sales metrics.

## 4. How To Show Business Impact

Strong impact language:

- Reduced manual reporting work
- Improved data trust
- Standardized metric definitions
- Enabled reorder decisions
- Detected data quality issues before reporting
- Supported faster sales and inventory analysis

If you do not have real production metrics, use realistic project outcomes without inventing numbers.

Example:

> Produced dashboard-ready marts for daily sales, product performance, and inventory reorder analysis.

## 5. Project Section Template

```text
Retail Sales Pipeline
- Built an ETL pipeline that reads retail CSV files, cleans sales/product/customer/inventory data, calculates gross sales, net sales, cost, and gross profit, and loads PostgreSQL tables.
- Added SQL quality checks for duplicates, missing references, negative quantities, and revenue sanity.
- Documented business problem, architecture, data model, setup, and expected outputs for portfolio review.
```

## 6. Junior Resume Focus

Emphasize:

- SQL
- Python
- Database loading
- Data cleaning
- Clear documentation
- Beginner projects with business framing

Avoid:

- Claiming senior tools without projects
- Listing every tool from the internet
- Saying "big data" without scale or use case

## 7. Mid-Level Resume Focus

Emphasize:

- Pipeline ownership
- Scheduling
- Data quality
- Warehouse models
- dbt or Airflow
- Production improvements
- Troubleshooting

Use verbs:

- Designed
- Built
- Automated
- Validated
- Optimized
- Documented
- Monitored

## 8. Senior Resume Focus

Emphasize:

- Architecture decisions
- Reliability
- Governance
- Cost optimization
- Mentoring
- Cross-functional impact
- Platform improvements

Use phrases:

- Defined architecture
- Established standards
- Improved reliability
- Reduced operational risk
- Enabled self-service analytics
- Governed data access

## 9. Resume Checklist

- [ ] Summary matches target role.
- [ ] Skills are grouped and honest.
- [ ] Projects include business context.
- [ ] Bullets show systems and outcomes.
- [ ] Data quality is mentioned.
- [ ] Production thinking is visible.
- [ ] No credentials or secrets are included.
- [ ] Links to GitHub and portfolio are present.
- [ ] Formatting is clean and easy to scan.
