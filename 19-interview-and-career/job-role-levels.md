# Job Role Levels

Data engineering titles vary by company. Use this guide to understand expectations, not as a strict rule.

## 1. Entry-Level Or Junior Data Engineer

Typical responsibilities:

- Write SQL queries
- Build small ingestion scripts
- Load files into databases
- Fix simple pipeline issues
- Add basic data quality checks
- Document tables and project steps

Expected skills:

- SQL joins, aggregations, and windows
- Python scripts with functions and logging
- Git basics
- PostgreSQL or similar database
- CSV, JSON, and API basics
- Basic ETL/ELT

Portfolio expectation:

- 2 to 4 small projects
- Clear README files
- At least one project with Python and SQL
- Evidence of validation and error handling

Interview focus:

- SQL
- Python fundamentals
- Data formats
- Basic pipeline design
- Explaining projects clearly

## 2. Mid-Level Data Engineer

Typical responsibilities:

- Own production pipelines
- Design warehouse tables and marts
- Build scheduled workflows
- Write tests and quality checks
- Troubleshoot failures
- Improve pipeline reliability and performance

Expected skills:

- Advanced SQL
- Data modeling
- Airflow or another orchestrator
- dbt or analytics engineering workflow
- Incremental loading
- Data quality strategy
- Cloud basics
- CI/CD basics

Portfolio expectation:

- 3 to 5 stronger projects
- One warehouse or dbt project
- One scheduled pipeline
- Data quality checks
- Architecture diagrams
- Production improvement notes

Interview focus:

- Pipeline design
- Data modeling
- Orchestration
- Data quality
- Debugging failures
- Business impact

## 3. Senior Data Engineer

Typical responsibilities:

- Design data platform architecture
- Set technical standards
- Review designs and mentor others
- Own reliability, cost, and governance tradeoffs
- Partner with business and product teams
- Lead large migrations or platform improvements

Expected skills:

- System design
- Warehouse and lakehouse architecture
- Spark or distributed processing when needed
- Streaming concepts
- Security and governance
- Observability
- Cost optimization
- Stakeholder communication

Portfolio expectation:

- Capstone-level project
- Strong architecture explanations
- Tradeoff analysis
- Security and cost considerations
- Monitoring and backfill strategy
- Clear business impact narrative

Interview focus:

- Architecture tradeoffs
- Scaling and reliability
- Data contracts
- Incident handling
- Leadership and communication
- Business outcomes

## 4. Analytics Engineer

Typical responsibilities:

- Build dbt models
- Define metrics
- Maintain marts
- Write tests and docs
- Support BI and reporting
- Partner with analysts and business users

Expected skills:

- SQL
- dbt
- Dimensional modeling
- Data quality tests
- Documentation
- BI awareness
- Metric governance

Portfolio expectation:

- dbt project
- Star schema or marts
- Tests and docs
- Metric dictionary
- Dashboard-ready outputs

## 5. Data Platform Engineer

Typical responsibilities:

- Build shared data infrastructure
- Manage orchestration, storage, compute, and permissions
- Improve developer experience
- Support reliability and observability
- Enable teams to build data products

Expected skills:

- Cloud infrastructure
- Deployment
- CI/CD
- IAM and security
- Observability
- Containerization
- Platform architecture

Portfolio expectation:

- Architecture-heavy project
- Deployment documentation
- Monitoring and security plan
- Cost and scalability discussion

## 6. How To Grow Between Levels

Junior to mid-level:

- Move from tasks to ownership.
- Learn orchestration and quality checks.
- Document assumptions and runbooks.
- Build projects that run repeatedly.

Mid-level to senior:

- Move from pipelines to systems.
- Make tradeoffs explicit.
- Improve reliability and cost.
- Mentor others through documentation and reviews.
- Connect technical design to business impact.
