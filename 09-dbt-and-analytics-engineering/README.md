# 09 - dbt And Analytics Engineering

## What This Section Covers

This section covers analytics engineering concepts such as modular SQL transformations, model layers, testing, documentation, lineage, metrics, and warehouse-centered development workflows.

## Why It Matters In Data Engineering

Analytics engineering helps teams make warehouse transformations reliable, documented, reusable, and reviewable. It bridges raw data pipelines and business-facing analytics.

## What You Should Be Able To Do

After studying this section, you should be able to:

- Organize SQL transformations into clear layers.
- Define staging, intermediate, and mart models.
- Add tests for important assumptions.
- Document models and columns.
- Understand lineage between source data and final metrics.
- Build analytics datasets with maintainable SQL.
