# Capstone Projects

Capstone projects combine the roadmap into complete, portfolio-ready systems. They should demonstrate business understanding, data modeling, pipeline design, quality strategy, production thinking, governance, and clear communication.

Each capstone is designed as a professional case study. A learner should be able to implement one capstone in stages, document tradeoffs, and present the final work to hiring managers, clients, or technical reviewers.

## Capstones

| Capstone | Focus | Best Portfolio Signal |
| --- | --- | --- |
| `retail-intelligence-platform` | End-to-end retail analytics | Sales, customer, inventory, and operations intelligence |
| `inventory-optimization-system` | Inventory decision support | Reorder logic, stock risk, supplier and demand analytics |
| `sme-data-platform` | Practical data platform for small businesses | Low-cost, maintainable, business-first architecture |
| `ai-business-analytics-assistant` | Governed AI analytics | Semantic layer, RAG, data API, and trusted AI answers |

## How To Use These Capstones

Start with the README as a project brief. Then build the system in phases:

1. Define the business problem and target users.
2. Create sample data and a data model.
3. Build the first working pipeline.
4. Add data quality checks.
5. Add analytics outputs.
6. Add production features.
7. Document tradeoffs, cost, governance, and future work.

The final portfolio artifact should include code, documentation, diagrams, sample outputs, and a short business impact explanation.
