# Intermediate Projects

Intermediate projects turn beginner skills into portfolio-ready systems. They focus on realistic business problems, layered architectures, data quality, documentation, and production improvement paths.

These projects are specifications and skeletons first. Full implementations can be added stage by stage as learners progress through SQL, Python, data warehousing, dbt, orchestration, and production engineering.

## Projects

| Project | Focus | Business Outcome |
| --- | --- | --- |
| `01-api-to-postgres-etl` | API extraction, pagination, PostgreSQL loading | Reliable sales or ecommerce data ingestion |
| `02-retail-sales-warehouse` | Dimensional modeling and marts | Trusted sales, product, customer, and inventory analytics |
| `03-inventory-reorder-pipeline` | Inventory analytics and business rules | Practical reorder recommendations |
| `04-dbt-retail-transformations` | dbt models, tests, docs, lineage | Governed analytics engineering workflow |
| `05-airflow-scheduled-pipeline` | Scheduling, retries, backfills, alerts | Automated daily retail reporting pipeline |

## How To Use This Section

Start by reading each project README as if it were a client brief. Identify the business problem, expected output, and data model before writing code. Then implement one project at a time, adding code, tests, sample data, diagrams, and screenshots as the project matures.

## Portfolio Standard

Each intermediate project should eventually include:

- A clear business problem
- A reproducible setup
- Source data or source data simulation
- Pipeline code or SQL
- Data model documentation
- Data quality checks
- Expected outputs
- Production improvement notes
- A short explanation of business impact
