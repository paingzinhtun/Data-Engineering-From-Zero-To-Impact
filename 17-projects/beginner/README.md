# Beginner Projects

Beginner projects are small, practical exercises that prove the core skills: reading source data, understanding a business problem, loading or querying data, validating outputs, and documenting the work.

Start here if you are new to Data Engineering or if you want a clean portfolio foundation before using heavier tools.

## Projects

| Project | Focus | Best For |
| --- | --- | --- |
| `01-csv-to-postgres` | CSV ingestion and database loading | First database loading project |
| `02-api-to-csv` | API extraction and JSON-to-CSV flow | Learning API basics |
| `03-sql-sales-analysis` | SQL business analysis | Practicing joins, aggregations, and reporting |
| `04-retail-sales-pipeline` | Beginner production-style ETL | Combining Python, SQL, PostgreSQL, Docker, tests, and quality checks |

## Recommended Order

1. Complete `01-csv-to-postgres`.
2. Complete `02-api-to-csv`.
3. Complete `03-sql-sales-analysis`.
4. Build and run `04-retail-sales-pipeline`.
5. Revisit the first three projects and improve their documentation using `18-templates/`.

## Completion Standard

Each beginner project should include:

- A clear business problem
- Source data description
- Setup instructions
- Expected output
- At least one validation or quality check
- A short explanation of business impact
