# Communities

Communities help you learn from real problems, ask better questions, and understand how data engineering is practiced in different companies.

## How To Use Communities Well

Before asking questions:

- Read the documentation.
- Search previous discussions.
- Create a small reproducible example.
- Explain what you tried.
- Include error messages and relevant code.
- Avoid sharing secrets, credentials, or private data.

## Community Etiquette

Good questions include:

- Clear context
- What you expected
- What happened
- Minimal example
- Relevant logs
- Tool versions

Poor questions:

- "It does not work" with no context
- Screenshots only
- Requests to complete assignments
- Sharing private data
- Asking for production advice without constraints

## Learning Habit

Once a week, read one real discussion about a pipeline failure, query performance issue, data quality problem, or architecture tradeoff. Summarize what failed, why it mattered, and how you would prevent it.
