# Courses

Courses are useful for structure, but they should not replace hands-on projects. A good course teaches concepts, provides practice, and helps you explain tradeoffs.

## Course Selection Criteria

Choose courses that include:

- Clear learning outcomes
- Hands-on projects
- SQL and Python practice
- Data modeling
- Data quality
- Production considerations
- Realistic business cases

Be cautious with courses that:

- Focus only on tool demos
- Skip fundamentals
- Use unrealistic toy examples
- Do not include projects
- Promise job readiness without portfolio work

## Beginner Courses To Look For

Search for:

- SQL for data analysis
- Python for data engineering
- Git and GitHub basics
- Linux command line basics
- PostgreSQL fundamentals
- APIs for beginners

Expected outcome:

- You can query databases, write Python scripts, read files, call APIs, and explain basic ETL.

## Intermediate Courses To Look For

Search for:

- Data warehousing
- Dimensional modeling
- dbt fundamentals
- Apache Airflow fundamentals
- Data quality testing
- Cloud data engineering basics

Expected outcome:

- You can build scheduled pipelines, warehouse models, dbt transformations, and quality checks.

## Advanced Courses To Look For

Search for:

- Apache Spark with PySpark
- Kafka streaming fundamentals
- Lakehouse architecture
- Cloud warehouse design
- Data platform architecture
- Data observability
- AI-ready data engineering

Expected outcome:

- You can discuss scale, cost, reliability, security, and tradeoffs.

## Free Learning Sources

Good places to search:

- Official documentation tutorials
- University open course materials
- Cloud provider learning portals
- Open-source project tutorials
- Conference talks from data engineering communities

## Paid Course Evaluation

Before paying for a course, check:

- Does it include projects?
- Are the tools current enough?
- Does it explain why choices are made?
- Does it teach debugging and failure handling?
- Does it include data quality?
- Can the final project become portfolio material?

## How To Combine Courses With This Repo

Use courses to learn specific skills, then apply them here:

- SQL course -> `02-sql-and-databases/`
- Python course -> `03-python-for-data-engineering/`
- Modeling course -> `04-data-modeling/`
- Airflow course -> `08-orchestration/`
- dbt course -> `09-dbt-and-analytics-engineering/`
- Spark course -> `10-big-data-processing/`
- Kafka course -> `11-streaming-data/`
- Cloud course -> `13-cloud-data-engineering/`

## Practical Rule

For every course you take, build or improve one project. Learning counts when you can use it.
