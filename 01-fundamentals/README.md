# 01 - Fundamentals

## What This Section Covers

This section covers the basic building blocks of data work: files, tables, records, fields, schemas, data types, keys, data formats, source systems, and common data quality problems.

## Why It Matters In Data Engineering

Every pipeline depends on a correct understanding of the data it handles. If learners do not understand schemas, formats, missing values, duplicate records, and business meaning, they will build fragile systems no matter which tools they use.

## What You Should Be Able To Do

After studying this section, you should be able to:

- Read and inspect CSV, JSON, Excel, and text-based data.
- Explain rows, columns, schemas, records, and keys.
- Identify missing, duplicated, inconsistent, and invalid data.
- Create a simple data dictionary.
- Connect data fields to real business processes.
